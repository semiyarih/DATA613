Shiny is an open source R package that provides an elegant and powerful web framework for building web applications using R. It creates interactive web applications.

Structure of a Shiny App

Shiny apps are contained in a single script called app.R. The script app.Rlives in a directory (for example, newdir/) and the app can be run with runApp(“newdir”).

app.R has three components:

  • a user interface object

The user interface (ui) object controls the layout and appearance of your app.

  • a server function

The server function contains the instructions that your computer needs to build your app.

  • a call to the shinyApp function

Finally the shinyApp function creates Shiny app objects from an explicit UI/server pair.

User Interface, ui

Here is the ui object for the Hello Shiny example.

library(shiny)

# Define UI for app that draws a histogram ----
ui <- fluidPage(

  # App title ----
  titlePanel("Hello Shiny!"),

  # Sidebar layout with input and output definitions ----
  sidebarLayout(

    # Sidebar panel for inputs ----
    sidebarPanel(

      # Input: Slider for the number of bins ----
      sliderInput(inputId = "bins",
                  label = "Number of bins:",
                  min = 1,
                  max = 50,
                  value = 30)

    ),

    # Main panel for displaying outputs ----
    mainPanel(

      # Output: Histogram ----
      plotOutput(outputId = "distPlot")

    )
  )
)

shinyApp Function

In your app.R file you will need to start with loading the Shiny package and end with a call to shinyApp by adding shinyApp(ui = ui, server = server). As soon as you saved this last part, the app will be active and you will see on right top corner f your script the icon Run App. Note, if you don’t see it , then you need to save your script first.

library(shiny)

Running an App

Important note:

  • Every Shiny app has the same structure: an app.R file that contains ui and server. You can create a Shiny app by making a new directory and saving an app.R file inside it. It is recommended that each app will live in its own unique directory.

There are a few ways you can run an app. For instance to run app.R we can do

  1. on app.R click on Run Appicon lcocated top right corner of your script.

  2. by giving the name of its directory to the function runApp. I have saved my app.R in a folder called a_shiny. So, I will type in my console the following

runApp("./a_shiny")
  1. If your file is saved in an R Script with different name than app.R, for instance hello_app.R, you need to provide path to the file (file name must be included)
runApp("./a_shiny/hello_app.R")
  • In this case, the name of app does not need to be ended with app.R. For instance you can have the script name as hello.R
runApp("./a_shiny/hello.R")

R session will be busy after running an App

Your R session will be busy while the Hello “Shiny app” is active, so you will not be able to run any R commands. R is monitoring the app and executing the app’s reactions. To get your R session back, hit escape or click the stop sign icon (found in the upper right corner of the RStudio console panel).
